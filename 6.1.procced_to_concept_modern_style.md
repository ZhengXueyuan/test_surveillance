你是一位高级前端工程师，正在为交易监控系统开发一个现代化仪表盘。请基于以下需求重写 'StatusBoard.vue' 组件：

## 设计目标
- 整体风格：深色主题（#0a1028 背景），科技感强，符合概念图风格
- 布局：左侧固定菜单栏 + 顶部导航栏 + 主内容区 + 底部状态栏
- 支持响应式（移动端适配）

## 功能要求
a. **左侧菜单栏**：
   - 固定宽度 200px
   - 导航项：Components（默认选中）、Files、Alerts、Settings
   - 使用图标+文字（用 font-awesome 或 SVG）

b. **顶部导航栏**：
   - 包含：Logo、搜索框、用户头像、通知铃铛、手动刷新按钮
   - 时间显示在右上角（格式：2026/02/23 18:06）

c. **主内容区**：
   - 显示组件列表（表格或卡片）
   - 每行包含：组件ID、心跳状态、文件更新、运行等级、整体健康
   - 支持按健康排序（异常在前）
   - 添加搜索框和“手动刷新”按钮

d. **底部状态栏**：
   - 显示：最后更新时间、组件总数、正常数、异常数
   - 如：“最后更新: 2026/02/23 18:06 | 共 3 个组件 | 正常: 1 | 异常: 2”

e. **状态可视化**：
   - 心跳：🟢 存活 / 🔴 宕机
   - 文件：🟢 正常 / 🔴 异常
   - 运行等级：🟢 合规 / 🟡 违规 / ❌ 未知
   - 整体健康：🟢 正常 / 🔴 异常

## 技术要求
- 使用 Vue 3 Composition API
- 不引入第三方 UI 库（仅用原生 CSS）
- 所有样式用 scoped CSS
- 使用 <script setup>
- 保持代码简洁、可读

## 输出要求
- 返回完整的 .vue 单文件组件代码；
- 不要解释，只输出代码并写入文件。
