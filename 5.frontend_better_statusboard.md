你是一位前端工程师，正在为交易监控系统开发状态面板。当前使用 Vue 3 Composition API + Vite。

请基于以下需求，重写 StatusBoard.vue 组件：

## 当前功能（需保留）
- 表格展示组件 ID、心跳状态、文件更新、运行等级、整体健康
- 每 10 秒自动轮询 http://localhost:8000/api/v1/status
- 使用颜色和 emoji 表示状态（🟢/🔴）
- 支持 CORS 代理（通过 /api 转发）

## 新增需求
a. **现代化 UI**：
   - 使用简洁卡片式表格（无边框，带 hover 效果）
   - 状态用彩色圆点 + 文字（🟢 正常 / 🔴 异常 / 🟡 警告）
   - 字体清晰，间距舒适

b. **增强信息展示**：
   - 心跳列：显示“存活”或“宕机”，并附带最后心跳时间（如 "2026-02-23 17:30:00"）
   - 文件列：hover 时显示完整文件路径和最后修改时间
   - 等级列：显示“合规”或“违规”，并标注实际/期望值

c. **交互优化**：
   - 添加搜索框，可按 component_id 过滤
   - 默认按“整体健康”排序（异常在前）
   - 添加“手动刷新”按钮

d. **健壮性**：
   - 如果某组件缺少 heartbeat/file_status/level_status，显示“—”
   - 网络错误时显示 toast 提示（用 alert 临时替代）

## 技术要求
- 只使用 Vue 3 + 原生 JS（不引入 Element Plus 等 UI 库）
- 使用 <script setup>
- 样式用 scoped CSS
- 保持代码简洁、可读

## 输出要求
- 返回完整的 .vue 单文件组件代码；
- 不要解释，只输出代码。
