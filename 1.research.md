你是一位资深全栈工程师，正在设计一套用于模拟交易系统的组件状态监控系统。系统包含前端、后端和多个模拟交易组件。请完成以下任务：

1. **定义三种监控维度的数据模型**：
   a. 组件运行状态（含进程存在性、心跳时间戳）；
   b. 文件更新状态（基于计划文件，定义输入/输出文件路径及其预期更新周期）；
   c. 组件运行等级（基于计划文件，定义每天不同时间段应处于的等级，1=启动, 2=连接, 3=部分功能, 4=完全正常）。

2. **设计两个 YAML 格式的计划文件模板**：
   - file_monitor_plan.yaml：描述每个组件依赖的输入文件和产生的输出文件，以及它们的预期更新 cron 表达式。
   - level_schedule.yaml：描述每个组件在一天中不同时间段应达到的运行等级。
   - 可以参考'file_monitor_plan_ref'.'yaml和level_schedule.yaml'

3. **描述整体系统架构**：
   - 所有组件通过 HTTP 向后端上报心跳；
   - 后端有一个中央状态存储（如 Redis）统一保存所有监控数据；
   - 一个独立的定时任务服务定期检查文件更新和运行等级合规性，并更新中央状态；
   - 前端通过 REST API 从中央状态读取实时监控视图；
   - 强调“数据面分离”：组件只写，前端只读，中央存储是唯一真相源。中央存储源隔离读写后自动处理大部分读写未对齐引起的错误；
   - 组件之间的结构请参考'framework_draft.md'

4. **描述网页结构**
   - 至少包含以下部分：顶部导航栏、侧边栏菜单、主内容区和底部状态栏
   - 布局可以参考'html_draft.png'

5. **输出要求**：
   - 用清晰的 Markdown 格式组织答案；
